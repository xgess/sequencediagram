// Example 13: Message Delays and Boundary Messages
// Demonstrates delay notation and incoming/outgoing boundary messages

title Delays and Boundary Messages

participant Client
participant Server
participant External

// Incoming boundary message (from outside the diagram)
[->Client:User input

// Normal synchronous messages
Client->Server:Request

// Delayed message (sloped line, N=2 delay units)
Server->(2)External:External API call

// Larger delay
External-->(3)Server:API response (slow)

// Return with delay
Server-->(1)Client:Response

// Outgoing boundary message (to outside the diagram)
Client->]:Notify user

// Combinations with different arrow types
[->Server:System event

Server->>(1)Client:Async update

Client-->]:Log event

// Lost message (ends with X)
Server-xExternal:Failed request
